Python vs Java

The first conceptual demo of code for each section is done in Python (unless specified).

General comparison of techniques

System.out.println() is essentially just a print statement in Python.

Layout

All statements in Python are separated by new lines whereas with Java (and most other languages) they are done with a semi colon (;).

Python

The layout of Python code is important as it is indentation based. Python determines what is within a if statement by reading the line with, for example, if on it, then going to the next indented line. It determines the end of the if block when it comes across code less indented than the previous line.

myNumber = 5
if myNumber > 10:
    print("Greater than 10")
else:
    print("Less than 10")

Java

Java (and most other programming languages) work instead with braces ({ and }) to specify what is within a block of code, indentation does not matter to the program, however it is still good practice to indent as you would in Python for readability.

int myNumber = 5;
if (myNumber < 10) {
    System.out.println("Greater than 10");
} else {
    System.out.println("Less than 10");
}

The following however would also work perfectly in Java, but is less readable (and you may lose marks on code for poor/inconsistent layout):

int myNumber = 5;
if (myNumber < 10) {System.out.println("Greater than 10")   ;
} else { System.out.println("Less than 10");}

Access Modifiers

Python doesn't really have this fully implemented, and I don't think you met it last year - so not going to compare Python here.

In Java there are 3 access modifiers, and not specifing one (so 4 ways to define the access of a variable)

public

The public keyword means the variable will be accessible from anywhere else in the program.

private

The private keyword means the variable will only beaccessible from the current class.

protected

The protected keyword means the variable will be accessible from the current package and anything within it (including subclasses - i.e classes that extend the class the variable is in - more later).

Specifiying no access modifier

Failing to specify a modifier means the variable can be access from the current class and package,

As a table:

As the table goes down, less and less places can access the variable
Access Modifier Same Class Same Package Subclass Other Packages
Public Y Y Y Y
Protected Y Y Y N
None Y Y N N
Private Y N N N

Variables

The data type of the variable in python is determined by the interpreter, so you do not specify a type. In Java you have to specify a type. The common types youâ€™ll be using are:

  • int
  • float
  • double - more accurate than float
  • String

Python

The way python declares variables is as so: 
myInt = 5
myFloat = 5.0
myString = "Hello"
Python doesn't really care and will just work it out based on context, and sometimes will convert between data types depending on what you try to do with it. This is allowed as Python can dynamically (meaning at run time) allocate memory at will, where as with Java you have to do this yourself.

Generally:

<var_name> = <value>

Java

Java requires you to specify a type for the variable: 
int myInt = 5;
float myFloat = 5.0;
String myString = "Hello";
This is so that the compiler knows how much memory to set aside for the variable.


Generally:

[access_modifier] <data_type> <var_name> = <value>;

Constants

Python doesn't support constants. A constant is a value in memory that cannot be changed.

Java doesn't directly support constants via a single keyword, however, the effect of a constant can be achieved as follows:

static final keywords

To tell the compiler you don't want the variable to change, you put static final before the data type of the constant: 
private static final int myInt = 5;
private static final float myFloat = 5.0;
private static final String myString = "Hello";
This prevents you from changing the values during run time.

final means that the variable declaration is final... i.e it can't be changed. Static is explained here.

Generally:

<access_modifier> static final <data_type> <const_name> = <value>;

Objects

Python has these, but not sure if you covered them - so not going to compare Python here.

Everything in Java has to be a class. A class is a 'blueprint' of what ever you are modelling with the class.
For example, you may have a class called Dog. When you create an instance of that class, you then have an object. So, a object is an instance of a class.

A class may have have properties about what you're modelling (i.e variables), as well as methods (functions) to perform tasks using these properties.

Typically properties within a class (something referred to as 'class members') are declared as private, and 'getter/setter' methods are used to retrieve and alter their values. This is so that an external object or method doesn't try to directly edit the variable, as the object will not be able to make sure the input is valid (where as a setter method can perform checks).

A 'Dog' Class

Here is a simple dog class, which we will be the example for the rest of this section for explanations: 
public class Dog {
    private String name;
    private int age;
    
    // Constructor, explained later in this section
    public Dog(String dogName, int dogAge) {
        name = dogName;
        age = dogAge;
    }
    
    // Getter, returns the name of the current dog object
    public String getName() {
        return name;
    }
    
    // Setter, sets the name of the current dog object
    public void setName(String newName) {
        name = newName;
    }
    
    // Getter, returns age of the current dog object
    public int getAge() {
        return age;
    }
    
    // Setter, sets the age of the current dog object
    public void setAge(int newAge) {
        age = newAge;
    }
}

Constructor

The constructor is a method (function) which is automatically called when an object is instantiated (created, explained at the end of this section).
The constructor can take parameters in order to set member variables (explained next), or to determine the value of member variables. The constructor can also do other tasks such as record how many 'Dog' (in our example) objects have been created.

Multiple constructors are allowed so long as have different parameters (types, amount or order of data types).

Member Variables

These are simply variable which relate and belong to a object. These are typically private as to prevent outside places modifiying their value.

Setters/Getters

Setters and getters are a way for us to retrieve and manipulate our member variables.

Methods

Methods are fuctions that belong to a class, as mentioned previously, Java forces you to use classes for everything, so all 'functions' are actually methods.

Creating an Instance of a Object

We can create an instance of a object from elsewhere in our code as follows: 
Dog myDog = new Dog("Dot", 3);
This is essentially a variable, where the 'data type' is the name of our class, we give this the name 'myDog', this is then set to a new Dog. This means that we are creating a new instance of our class (now called a object because we have used the blue print so to speak). In the brackets, we pass the parameters that our contructor requires.

Different instances of a class have different values for their member variables (name and age in our example).

Access Object Methods

To access methods within our object we use dot notation, as shown: 
System.out.println(myDog.getName());
This line would print out the dog name onto our screen (as getName() returns a String, the name of the dog).

Static

References the 'Dog' class example from the previous topic (here).

The static keyword allows us to access variable within a class, without creating an object of that class. This is why the main method is a static method, as Java needs a way to call the method, as no objects have been created yet.

static also allows us to have variables shared across all instances of a particular class.

Adding a static member variable to our previous 'Dog' class

New lines have been marked with a comment saying 'NEW': 
public class Dog {
    public static int numberOfDogs = 0; // NEW
    
    private String name;
    private int age;
    
    // Constructor, explained later in this section
    public Dog(String dogName, int dogAge) {
        name = dogName;
        age = dogAge;
        
        // Increment the numberOfDogs variable
        numberOfDogs++; // NEW
    }
    
    // Setts and Getters here...
}
All dog objects will have access to the same variable, i.e they all access the exact same piece of memory for that variable.

If Statements

Concept of if statements is the same. 
if <condition>:
    // do this
else if <another_condition>:
    // do this instead
else:
    // do this if others all false

The below examples are from the layout section so are already explained, but here again:

Python

Python uses indentation, brackets around condition are optional.


myNumber = 5
if myNumber > 10:
    print("Greater than 10")
else:
    print("Less than 10")

Java

Java uses { and } to contain the blocks of code, i.e the body of the if statement. Brackets around the condition are compulsory.

int myNumber = 5;
if (myNumber < 10) {
    System.out.println("Greater than 10");
} else {
    System.out.println("Less than 10");
}

While loops

Concept of while loops is the same. 
while <condition>
    // do this
    // can break out of while loop using 'break' (for both Python and Java)

Python

Python uses indentation, brackets around condition are optional.


counter = 0
while counter < 5:
    print(counter)
    counter += 1

Java

Java uses { and } to contain the blocks of code, i.e the body of the while loop. Brackets around the condition are compulsory.

int counter = 0;
while (counter < 5) {
    System.out.println(counter);
    counter++;
}

For loops

Concept of for loops is the same, although they are defined different. 
for <temp_variable> in range(<start>, <end>, <step>):
    // do this

We go from the start variable up to (but not including) the end variable with an increment of step so range(1, 11, 2) gives an output of 1, 3, 5, 7, 9 (notice no 11). Both the example below use this start, end and step and produce the same result.

Python

Python uses indentation, brackets around for condition (the i in range(1, 11, 2) part) are optional.


for i in range(1, 11, 2):
    print(i)

Java

Java uses { and } to contain the blocks of code, i.e the body of the for loop. Brackets around the condition are compulsory.

Java also has start, end and step, just no range function. The header for a for loop is split into the same for parts, specifing a starting a value, some value to cause it to stop and how to change after each iteration, each separated by a ;. General syntax is:

for (<datatype> <var> = <start>; <condition>; <step>)

Note: <condition> and <step> are given in terms of <var>, see below example.

You need to declare the variable i (call it anything) in the first section, in the second section is the condition (i.e what causes the end) and finally the step or how to change after each iteration.

for (int i = 1; i < 11; i += 2) {
    System.out.println(i);
}

For vs While loops (Comparison)

The same thing can be accomplished with both a for loop and a while loop. A for loop is typically used when you know the range you want to iterate through whereas a while loop you use when you don't know or can't find out via something like user input. The comparison is done using Java code.

While loops

int counter = 0;
while (counter < 5) {
    System.out.println(counter);
    counter++;
}

A separate counter variable has to be declared and initialised outside of the loop, you must remember to increment it at the end of the while loop.

For loops

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

The counter variable initialisation, condition and incrementation code is all on one line and looks tidier.

Functions/Methods

Concept of functions is the same, although they are defined slightly different. 
def <function_name>(<any_arguments_comma_separated>) :
    // do this
    // maybe return <value> (not neccesssary - function may just 'do' something i.e print something)


A method is a function that belongs to a class, so in Java which forces you to have a class for everything, all 'functions' are actually methods.

Python

Python uses indentation. Python knows you want to create a function as you typed def (define), the next bit is the name of the function, followed by any arguments the function may take in brackets, separated by commas. The function body is then indented. If the function returns something, then the return will give the result back to the caller.

Example of function that has return:

def add(num1, num2):
    result = num1 + num2
    return result

Example of function that has return:

def sayName(name):
    print(name)

Java

Java uses { and } to contain the blocks of code, i.e the body of the function. Brackets around the condition are compulsory. Functions in Java require a type, this is the kind of data that the function will return, this type is the same as the one listed in the variables section, or void for a function which does not return anything (i.e a function that does a task like print something). General syntax:

<access_modifier> <data_type> <funct_name>(<data_type> <name_within_funct>, [more]) {
    // do this
    // maybe return <value>;
}

Example of function that has return:

private float add(float num1, float num2) {
    float result = num1 + num2;
    return result;
}

Example of function that no return:

private void sayName(String name) {
    System.out.println(name);
}
